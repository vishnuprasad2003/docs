## Verification Agent Overview

Verification Agents are responsible for reviewing and verifying loan data quality, ensuring accuracy, completeness, and regulatory compliance before loan pools and deals proceed to market.

## Key Responsibilities

### Data Quality Review
- Review uploaded loan tapes
- Verify data accuracy
- Check data completeness
- Validate field mapping

### Compliance Verification
- Ensure regulatory compliance
- Verify required documentation
- Check industry standards
- Maintain audit trail

### Quality Approval
- Approve high-quality pools
- Reject pools with issues
- Request corrections when needed
- Document decisions

## Verification Process

### Accessing Pools for Review

1. Navigate to **Verification Queue**
2. View pools awaiting review
3. Select pool to review
4. Access loan tape data

### Review Checklist

**Data Quality Checks**:
- [ ] All required fields present
- [ ] Data types are correct
- [ ] Values within acceptable ranges
- [ ] No duplicate records
- [ ] Unique loan identifiers

**Compliance Checks**:
- [ ] Regulatory requirements met
- [ ] Industry standards followed
- [ ] Documentation complete
- [ ] Audit trail maintained

### Making Decisions

**Approve Pool**:
- Click **Approve** button
- Add approval notes
- Generate verification certificate
- Status changes to **Verified**

**Reject Pool**:
- Click **Reject** button
- Provide detailed rejection reason
- Specify required corrections
- Status changes to **Rejected**

**Request Changes**:
- Request specific corrections
- Provide detailed feedback
- Maintain under review status
- Issuer can resubmit after corrections

## Verification Certificate

Upon approval:
1. Certificate automatically generated
2. DocuSign sent for signature
3. Certificate stored in IPFS
4. Blockchain record created

**Certificate Contents**:
- Verification agent identity
- Verification date and time
- Pool details and identification
- Quality metrics and scores
- Approval conditions (if any)

## Best Practices

<Accordion>
<AccordionItem title="Thorough Review">
Review all loan data thoroughly before approval. Incomplete review compromises data quality.
</AccordionItem>

<AccordionItem title="Clear Communication">
Provide clear, detailed feedback when requesting corrections. Specific feedback helps issuers make accurate fixes.
</AccordionItem>

<AccordionItem title="Documentation">
Document all verification activities and decisions. Maintain complete audit trail for compliance.
</AccordionItem>

<AccordionItem title="Timeliness">
Process verification requests promptly. Delays impact issuer timelines and market opportunities.
</AccordionItem>
</Accordion>
