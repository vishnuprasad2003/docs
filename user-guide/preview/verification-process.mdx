## Verification Process Overview

Verification agents review uploaded loan tapes to ensure data quality, accuracy, and compliance with standards before approval for market.

## Verification Workflow

### Step 1: Verification Request

Issuer submits pool for verification:
- Pool marked for verification
- Verification agent notified
- Access granted to pool data

### Step 2: Verification Agent Review

<Image src="/images/servicerUploadLoanTapesProcesses.png" title="Verification Agent Dashboard" alt="Verification agent review process" />

Verification agents access pool for review:

**Review Checklist**:
- Data completeness check
- Data accuracy verification
- Field mapping validation
- Compliance review
- Quality assessment

### Step 3: Verification Actions

**Approve Pool**:
- Click **Approve**
- Add approval notes
- Status changes to **Verified**
- NFT minting process begins

**Reject Pool**:
- Click **Reject**
- Provide rejection reason
- Status changes to **Rejected**
- Issuer notification sent

**Request Changes**:
- Request specific corrections
- Provide detailed feedback
- Status remains **Under Review**
- Issuer updates and resubmits

## Verification Criteria

### Data Quality Standards

**Required Checks**:
- All required fields present
- Data types correct
- Values within acceptable ranges
- No duplicate records
- Unique loan identifiers

### Compliance Verification

**Compliance Checks**:
- Regulatory requirements met
- Industry standards followed
- Documentation complete
- Audit trail maintained

### Quality Assessment

**Quality Metrics**:
- Data completeness rate
- Error rate
- Compliance score
- Overall quality rating

## Verification Certificate

### Certificate Generation

Upon approval:
1. Verification certificate generated
2. Signed via DocuSign
3. Stored in IPFS
4. Blockchain record created

**Certificate Contents**:
- Verification agent name
- Verification date
- Pool details
- Quality metrics
- Certificate ID

## Marketplace Submission

### Submission to Market Maker

After verification:
1. Pool becomes available to market makers
2. Market maker reviews pool
3. Market maker approves or rejects
4. Approved pools ready for investment
