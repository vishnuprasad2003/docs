## Monthly Reporting Overview

Monthly reporting provides stakeholders with comprehensive performance data about the loan portfolio within the credit facility. As a Servicer, you're responsible for submitting accurate and timely reports.

---

## Reporting Schedule

| Milestone | Timing |
|-----------|--------|
| **Monthly Upload Deadline** | Last business day of each month |
| **Data Cut-off** | End of reporting month |
| **Processing Time** | 2-3 business days |
| **Report Distribution** | 5th business day of following month |

<Note>
**Consistency is Key:** Regular, on-time submissions help all parties make informed decisions about the credit facility.
</Note>

### Data Quality Standards

Your reports should meet these standards:

| Standard | Requirement |
|----------|-------------|
| **Completeness** | All required fields populated |
| **Accuracy** | Data is correct and current |
| **Consistency** | Matches previous month's methodology |
| **Timeliness** | Submitted on schedule |

## Report Types

### Performance Report

The primary monthly report includes:

<Accordion title="Performance Report Contents" icon="file-chart-line" defaultOpen>
- **Executive Summary** - High-level overview of portfolio performance
- **Key Performance Indicators** - Delinquency rates, defaults, recoveries
- **Financial Highlights** - Cash flow, balances, rate changes
- **Risk Assessment** - Current risk profile and changes
- **Outlook** - Forward-looking commentary
</Accordion>

### Compliance Report

For regulatory and internal compliance:

<Accordion title="Compliance Report Contents" icon="shield-check">
- **Regulatory Status** - Compliance with applicable regulations
- **Audit Findings** - Results of any audits
- **Risk Assessment** - Risk-related compliance items
- **Corrective Actions** - Steps taken to address any issues
</Accordion>

## Report Generation Process

### Step 1: Data Collection

<Steps>
  <Step title="Gather Loan Data">
    Collect loan performance data from your loan management system.
  </Step>
  <Step title="Pull Financial Data">
    Gather financial transaction data for the reporting period.
  </Step>
  <Step title="Collect Supporting Data">
    Include any external market data or benchmarks if required.
  </Step>
</Steps>

### Step 2: Data Processing

<Steps>
  <Step title="Validate Data">
    Check all data for accuracy and completeness.
  </Step>
  <Step title="Aggregate Information">
    Combine data from multiple sources into report format.
  </Step>
  <Step title="Calculate Metrics">
    Perform required calculations (rates, percentages, totals).
  </Step>
  <Step title="Quality Check">
    Review for accuracy before finalizing.
  </Step>
</Steps>

### Step 3: Report Distribution

<Steps>
  <Step title="Get Approval">
    Have reports reviewed and approved by management.
  </Step>
  <Step title="Distribute">
    Submit reports through the platform and to stakeholders.
  </Step>
  <Step title="Archive">
    Store copies for future reference and audit purposes.
  </Step>
</Steps>

## Key Performance Metrics

### Primary KPIs

| Metric | Description |
|--------|-------------|
| **Delinquency Rate** | % of loans with late payments |
| **Default Rate** | % of loans that have defaulted |
| **Recovery Rate** | % recovered from defaulted loans |
| **Servicing Efficiency** | Operational performance metrics |

### Trend Monitoring

Regularly compare your metrics:

- **Month-over-month** - How did things change this month?
- **Year-over-year** - How does this compare to last year?
- **Against benchmarks** - How do we compare to industry standards?
- **Against targets** - Are we meeting our goals?

## Troubleshooting

<Accordion title="Data Doesn't Match Previous Month" icon="not-equal">
**Possible Causes:**
- Methodology change
- Data source change
- Calculation error

**What to Do:**
- Document the reason for variance
- Note any methodology changes
- Verify calculations are correct
</Accordion>

<Accordion title="Missing Data" icon="file-excel">
**Possible Causes:**
- Source system issues
- Integration problems
- Data not available yet

**What to Do:**
- Check source systems
- Contact IT if integration issues
- Note any missing data in report
</Accordion>

<Accordion title="Report Deadline at Risk" icon="clock">
**What to Do:**
- Communicate early with stakeholders
- Prioritize critical data elements
- Provide preliminary report if needed
- Document reason for delay
</Accordion>
