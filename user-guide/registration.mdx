## User Registration Process

The Intain Markets platform supports multiple user roles, each with specific registration requirements and capabilities. Follow this guide to register your account and get started with the credit facility platform.

## Supported User Roles

### **Issuer (Borrower)**
- Companies seeking credit facility funding
- Can create term sheets and funding requests
- Manages loan collateral and documentation

### **Facility Agent (Underwriter)**
- Financial institutions providing credit facilities
- Reviews and approves term sheets and funding requests
- Manages master commitments and lender groups

### **Lender (Investor)**
- Investors providing funding for credit facilities
- Reviews investment opportunities
- Confirms fund transfers and monitors performance

### **Servicer**
- Loan servicing companies
- Manages loan portfolios and reporting
- Uploads monthly loan tapes and performance data

## Registration Steps

### 1. **Initial Registration**

Navigate to the registration page and provide the following information:

```json
{
  "EmailAddress": "user@company.com",
  "UserRole": "Issuer",
  "FirstName": "John",
  "LastName": "Doe",
  "Password": "SecurePassword123!",
  "OrganizationName": "ABC Corporation",
  "PhoneNumber": "+1-555-0123"
}
```

**Required Fields:**
- **Email Address**: Valid business email address
- **User Role**: One of the supported roles (Issuer, Facility Agent, Lender, Servicer)
- **First Name**: Legal first name
- **Password**: Strong password (minimum 8 characters, mixed case, numbers, special characters)
- **Organization Name**: Legal business name

### 2. **Email Verification**

After registration, you'll receive a verification email:
- Click the verification link in your email
- Complete the email verification process
- Your account will be activated upon verification

### 3. **KYC/AML Verification**

Depending on your role, you may need to complete additional verification:

#### **For Issuers:**
- Business registration documents
- Financial statements
- Credit history verification
- Collateral documentation

#### **For Facility Agents:**
- Financial institution license
- Regulatory compliance certificates
- Underwriting authority documentation

#### **For Lenders:**
- Investment authority documentation
- Financial capacity verification
- Regulatory compliance certificates

#### **For Servicers:**
- Servicing license/certification
- Operational capacity documentation
- Technology infrastructure verification

### 4. **Blockchain Registration**

The platform automatically registers your account on the blockchain:
- User data is stored in IPFS (InterPlanetary File System)
- Blockchain record is created with your unique user ID
- Organization database is created (for Issuers)

### 5. **Account Activation**

Once all verifications are complete:
- Your account is activated
- You receive login credentials
- Access to role-specific features is granted

## Authentication Process

### **Login Requirements**

```json
{
  "EmailId": "user@company.com",
  "Password": "EncryptedPassword",
  "Role": "Issuer"
}
```

### **Authentication Flow**

1. **Credential Validation**
   - Email format validation
   - Password strength verification
   - Role verification

2. **Blockchain Verification**
   - User data retrieved from blockchain
   - Status verification (Active/Inactive)
   - Role confirmation

3. **IPFS Data Retrieval**
   - User profile data retrieved from IPFS
   - Organization information loaded
   - Preferences and settings applied

4. **JWT Token Generation**
   - Secure JWT token created
   - Token includes user ID, role, and organization ID
   - Token expires after 1 hour

5. **Session Management**
   - User session established
   - Role-based permissions applied
   - Dashboard data loaded

## Password Requirements

Your password must meet the following criteria:
- **Minimum 8 characters**
- **At least one uppercase letter**
- **At least one lowercase letter**
- **At least one number**
- **At least one special character**
- **Cannot be a common password**

## Security Features

### **Data Encryption**
- All passwords are encrypted using AES encryption
- Sensitive data is encrypted in transit and at rest
- Private keys are securely stored

### **Blockchain Security**
- User data is stored on immutable blockchain
- IPFS provides decentralized storage
- Cryptographic hashing ensures data integrity

### **Session Security**
- JWT tokens with expiration
- Secure session management
- Automatic logout on inactivity

## Troubleshooting

### **Common Issues**

#### **Registration Fails**
- Check email format and availability
- Ensure password meets requirements
- Verify all required fields are completed
- Check internet connection

#### **Email Verification Issues**
- Check spam/junk folder
- Verify email address is correct
- Request new verification email
- Contact support if issues persist

#### **Login Problems**
- Verify email and password
- Check if account is activated
- Ensure correct role is selected
- Clear browser cache and cookies

#### **KYC/AML Delays**
- Ensure all documents are clear and legible
- Provide complete documentation
- Follow up with support team
- Check document requirements
