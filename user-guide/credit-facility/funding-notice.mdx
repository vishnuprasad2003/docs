# Funding Notice & Token Management

Funding notices are automatically generated when funding requests are approved. This guide covers the complete process from generation to fund distribution.

## Overview

The funding notice process includes:

1. **Auto-Generation** - Created automatically when funding request is approved
2. **DocuSign Process** - Borrower signs funding notice electronically
3. **FT Token Creation** - Tokens created for borrower after DocuSign
4. **Token Transfer Approval** - Borrower approves token transfers to lenders
5. **Lender Distribution** - Individual lenders confirm fund transfers

## Auto-Generation Process

### Trigger Conditions

Funding notices are automatically created when:

- Funding request status changes to **APPROVED**
- Master commitment status is **ACTIVE**
- All required validations pass

### Generation Details

**Automatic Process:**
1. System generates unique funding notice ID
2. Copies data from approved funding request
3. Calculates token distribution for each lender
4. Sets initial status to **PENDING_ESIGN**
5. Prepares DocuSign envelope

**Pre-populated Data:**
- Request amount from funding request
- Master commitment details
- Lender group information
- Token distribution calculations

## DocuSign Process

### Signing Workflow

1. **Envelope Creation** - DocuSign envelope automatically created
2. **Borrower Notification** - Email sent to borrower for signing
3. **Electronic Signature** - Borrower completes DocuSign process
4. **Status Update** - Funding notice status: **ESIGN_COMPLETED**
5. **FT Token Creation** - Tokens automatically created

### Signing Requirements

**Borrower Must:**
- Complete DocuSign authentication
- Review funding notice details
- Provide electronic signature
- Confirm understanding of terms

## FT Token Management

### Token Creation

After DocuSign completion:

1. **Contract Deployment** - FT contract deployed to blockchain
2. **Token Minting** - Tokens minted for borrower
3. **Distribution Setup** - Lender token allocation prepared
4. **Status Tracking** - Individual lender status initialized

### Token Distribution

**Per-Lender Tracking:**
Each lender has individual status tracking:

- **tokensAllocated** - Amount of tokens allocated
- **mintingStatus** - Token transfer status (pending/completed)
- **lenderApprovalStatus** - Lender approval (PENDING/APPROVED/REJECTED)
- **amountTransferred** - Fund transfer confirmation (pending/transferred)

## Borrower Actions

### Approve Token Transfer

**Purpose:** Make funding notice visible to lenders

**Process:**
1. Navigate to funding notice details
2. Review token distribution
3. Click **"Approve Token Transfer"**
4. Confirm transaction

**Result:**
- Status changes to **ISSUED**
- Funding notice becomes visible to lenders
- Lenders can confirm fund transfers

### Token Transfer Approval

**Requirements:**
- Funding notice status: **ESIGN_COMPLETED**
- FT tokens created successfully
- Borrower wallet connected

**Steps:**
1. Connect wallet to platform
2. Review token distribution details
3. Approve token transfer privilege
4. Confirm blockchain transaction

## Lender Actions

### Fund Transfer Confirmation

**Process:**
1. Lender receives notification of funding notice
2. Reviews funding notice details
3. Confirms fund transfer completion
4. Updates individual status

**Status Updates:**
- **amountTransferred**: pending → transferred
- **lenderApprovalStatus**: PENDING → APPROVED

### Individual Lender Tracking

## Status Workflow

### Complete Status Flow

```
Funding Request APPROVED → Auto-Generate Funding Notice → PENDING_ESIGN
                                    ↓
                            DocuSign for Funding Notice
                                    ↓
                            ESIGN_COMPLETED (FT tokens created)
                                    ↓
                            Borrower Approves Token Transfer
                                    ↓
                            ISSUED (visible to lenders)
                                    ↓
                            Individual Lenders Confirm Fund Transfer
                                    ↓
                            Each lender's status updated in tokenDistribution
```

### Status Descriptions

- **PENDING_ESIGN** - Auto-generated, ready for DocuSign
- **ESIGN_COMPLETED** - DocuSign completed, FT tokens created
- **ISSUED** - Borrower approved token transfer, visible to lenders

## Token Distribution Details

### Calculation Method

Token allocation calculated based on:

- **Lender Voting Percentage** - From master commitment
- **Request Amount** - From funding request
- **Proportional Distribution** - Each lender gets their percentage

**Formula:**
```
Lender Tokens = (Lender Voting % / 100) × Request Amount
```

### Distribution Tracking

**Per-Lender Information:**
- **lenderGroupId** - Unique identifier
- **lenderOrgId** - Organization identifier
- **lenderName** - Lender organization name
- **votingPercentage** - Voting percentage from master commitment
- **commitmentAmount** - Lender's commitment amount
- **tokensAllocated** - Calculated token allocation
- **tokenContractAddress** - FT contract address
- **mintedAt** - Token minting timestamp
- **transactionHash** - Blockchain transaction hash
